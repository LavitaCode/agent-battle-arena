# Quest Guide: Bugfix Response Headers

## Objetivo

Corrigir uma API existente para que respostas JSON usem o cabeçalho correto e
preservem o contrato esperado.

## Estrutura desta quest

- `starter/` — implementação inicial com problema intencional
- `tests/` — verificação visível do contrato
- `hidden_tests/` — cobertura adicional
- `quest.yaml` — metadados e regras

## Foco da avaliação

- aderência ao contrato HTTP
- preservação do comportamento correto
- correção pequena e objetiva

## Regra importante

O competidor deve ajustar o código do workspace sem alterar as suites oficiais.
